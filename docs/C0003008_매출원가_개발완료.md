# C0003008 매출원가 화면 개발 완료

## 📋 개요
**메뉴**: C0003000 제조매출원가 > C0003008 매출원가  
**화면 유형**: 데이터 생성 화면 (조회 화면 아님)  
**목적**: STCO 출고금액을 거래처별 매출수량 비율로 배부하여 DOI_SLCO 테이블에 저장  
**개발일**: 2025-11-11

---

## 🎯 핵심 로직

### 매출원가 계산식 (UP_DOI_SALE_COST 프로시저)
```
매출원가(거래처별) = 출고금액(OUT_AMT) × (거래처별 매출수량 / 모델별 총 매출수량)
```

### 출고수량 배부식
```
출고수량(거래처별) = 총 출고수량(OUT) × (거래처별 매출수량 / 모델별 총 매출수량)
```

### 2. Backend (Stored Procedure)
- **파일**: `src/원가SQL/UP_DOI_SALE_COST.sql`
- **프로시저명**: `UP_DOI_SALE_COST`
- **파라미터**: `@YYYYMM`, `@SITE`

#### 프로시저 로직 구조
```sql
1. DOI_SLCO 기존 데이터 삭제 (YYYYMM, SITE 기준)

2. 매출원가 데이터 생성 및 INSERT
   - STCO (DOI_STOCK_COST): 모델별/원가항목별 출고 정보
   - SALE (DOI_SALE_RESC): 거래처별 매출 정보  
   - SALE_TOTAL: 모델별 총 매출수량 (배부 기준)
   
3. INSERT INTO DOI_SLCO
   - 출고수량 = AVG(OUT) × (거래처 매출수량 / 총 매출수량)
   - 출고금액 = SUM(OUT_AMT) × (거래처 매출수량 / 총 매출수량)
   
4. retMessage 형식으로 실행 결과 반환
   - [START] / [INFO] / [FINISH] / [ERROR] 태그
```

#### 주요 테이블
1. **DOI_STOCK_COST** (읽기): 모델별/원가항목별 출고금액 소스
2. **DOI_SALE_RESC** (읽기): 거래처별 매출수량 (배부 기준)
3. **DOI_SLCO** (쓰기): 거래처별 매출원가 결과 저장

#### INT Overflow 해결
- `CAST(stco.[OUT] AS BIGINT) * CAST(sale.sale_qty AS BIGINT)` 
- 큰 숫자 곱셈 시 BIGINT로 캐스팅하여 오버플로우 방지

## ✅ 검증 포인트

### 1. 데이터 생성 건수 확인
```sql
-- 202507/HQ 예상 결과: 69건
SELECT COUNT(*) AS 총건수
FROM DOI_SLCO
WHERE YYYYMM = '202507' AND SITE = 'HQ';
```

### 2. 출고금액 배부 검증
```sql
-- 같은 모델/구분/원가항목의 거래처별 합계 = STCO 출고금액
SELECT 
    s.MODEL, s.구분, s.EXPEN_SEL,
    SUM(s.OUT_AMT) AS 거래처별합계,
    c.STCO출고금액,
    ABS(SUM(s.OUT_AMT) - c.STCO출고금액) AS 차이
FROM DOI_SLCO s
INNER JOIN (
    SELECT MODEL, 구분, EXPEN_SEL, SUM(OUT_AMT) AS STCO출고금액
    FROM DOI_STOCK_COST
    WHERE YYYYMM = '202507' AND SITE = 'HQ' AND SEL_CODE = 'ACTUAL'
    GROUP BY MODEL, 구분, EXPEN_SEL
) c ON s.MODEL = c.MODEL 
   AND s.구분 = c.구분 
   AND s.EXPEN_SEL = c.EXPEN_SEL
WHERE s.YYYYMM = '202507' AND s.SITE = 'HQ'
GROUP BY s.MODEL, s.구분, s.EXPEN_SEL, c.STCO출고금액
HAVING ABS(SUM(s.OUT_AMT) - c.STCO출고금액) > 1;
-- 결과: 0건 (반올림 오차 ±1원 허용)
```

### 3. 매출수량 비율 검증
```sql
-- 거래처별 배부 비율 = 거래처 매출수량 / 모델 총 매출수량
SELECT 
    s.MODEL, s.구분, s.거래처,
    r.매출수량 AS 거래처매출수량,
    t.총매출수량 AS 모델총매출수량,
    ROUND(r.매출수량 * 100.0 / t.총매출수량, 2) AS 계산배부율,
    ROUND(s.OUT_AMT * 100.0 / SUM(s.OUT_AMT) OVER (PARTITION BY s.MODEL, s.구분, s.EXPEN_SEL), 2) AS 실제배부율
FROM DOI_SLCO s
INNER JOIN DOI_SALE_RESC r 
    ON s.거래처 = r.거래처 
   AND s.MODEL = r.품명
   AND r.YYYYMM = s.YYYYMM
   AND r.SITE = s.SITE
INNER JOIN (
    SELECT 품명, 
           CASE WHEN RIGHT(품번, 1) = 'D' THEN '개발' ELSE '양산' END AS 구분,
           SUM(수량) AS 총매출수량
    FROM DOI_SALE_RESC
    WHERE YYYYMM = '202507' AND SITE = 'HQ'
    GROUP BY 품명, CASE WHEN RIGHT(품번, 1) = 'D' THEN '개발' ELSE '양산' END
) t ON s.MODEL = t.품명 AND s.구분 = t.구분
WHERE s.YYYYMM = '202507' AND s.SITE = 'HQ';
-- 계산배부율 ≈ 실제배부율 확인
```

## 🐛 알려진 이슈 및 해결

### 1. ❌ INT Overflow (해결됨)
**증상**: 
```
CAST(ROUND(stco.[OUT] * sale.sale_qty * 1.0 / sale_total.total_sale_qty, 0) AS INT)
-- Arithmetic overflow error
```

**원인**: OUT * sale_qty 곱셈이 INT 최대값(2,147,483,647) 초과

**해결**: 
```sql
CAST(ROUND(CAST(stco.[OUT] AS BIGINT) * CAST(sale.sale_qty AS BIGINT) * 1.0 
    / sale_total.total_sale_qty, 0) AS INT)
```

---

### 2. ❌ 날짜 컬럼 불일치 (해결됨)
**증상**: STCO는 있는데 SALE JOIN 결과 0건

**원인**: 
```sql
-- 잘못된 방식
WHERE SUBSTRING(거래명세서일, 1, 6) = @YYYYMM
-- '2025-07-01' → SUBSTRING → '2025-0' (6자리 아님!)
```

**해결**: 
```sql
-- 올바른 방식
WHERE yyyymm = @YYYYMM
-- DOI_SALE_RESC.yyyymm 컬럼 직접 사용
```

---

### 3. ❌ retMessage vs retmessage (해결됨)
**증상**: "Cannot read properties of undefined (reading 'retMessage')"

**원인**: SQL은 대소문자 구분 안하지만 Vue는 구분함
- 프로시저: `SELECT @Message AS retMessage`
- Vue: `resp[0][0].retMessage` ❌

**해결**:
```javascript
// Vue에서 소문자로 파싱
if (resp && resp[0] && resp[0][0] && resp[0][0].retmessage) {
  this.resultMessage = resp[0][0].retmessage;  // ✅ 소문자
}
```
