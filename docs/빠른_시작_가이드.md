## 🛠️ 3단계: 우선순위별 데이터 수정

### 3.1 음수 재고 해결 (우선순위 1)

```sql
-- 1. 음수 재고 목록 확인
SELECT YYYYMM, DW_SITE, 도우모델, MODEL_N_TYPE, BOH_MONTH, EOH_MONTH
FROM DOI_PROD_SUBUL
WHERE EOH_MONTH < 0
ORDER BY YYYYMM, DW_SITE, 도우모델;

-- 2. 원인 분석
-- A) 전월 데이터 오류 → 전월 EOH 수정 후 재계산
-- B) 출고 과다 → 생산 실적 재확인
-- C) 투입 누락 → DW_PROD_* 원천 데이터 확인

-- 3. 수정 예시 (물리적 실사 후)
UPDATE DOI_PROD_SUBUL
SET EOH_MONTH = 100  -- 실사 수량
WHERE YYYYMM = '202510'
  AND 도우모델 = 'MODEL_A'
  AND DW_SITE = 'HQ';
```

### 3.2 생산수불 수식 오류 수정 (우선순위 2)

```sql
-- 1. 불일치 건 확인
SELECT 
    YYYYMM, 도우모델,
    BOH_MONTH, IN_MONTH, BONUS_MONTH, OUT_MONTH, LOSS_MONTH, NG_MONTH,
    EOH_MONTH AS 현재EOH,
    (BOH_MONTH + IN_MONTH + BONUS_MONTH - OUT_MONTH - LOSS_MONTH - NG_MONTH) AS 계산EOH,
    EOH_MONTH - (BOH_MONTH + IN_MONTH + BONUS_MONTH - OUT_MONTH - LOSS_MONTH - NG_MONTH) AS 차이
FROM DOI_PROD_SUBUL
WHERE ABS(EOH_MONTH - (BOH_MONTH + IN_MONTH + BONUS_MONTH - OUT_MONTH - LOSS_MONTH - NG_MONTH)) > 0.01;

-- 2. 자동 재계산
UPDATE DOI_PROD_SUBUL
SET EOH_MONTH = BOH_MONTH + IN_MONTH + BONUS_MONTH - OUT_MONTH - LOSS_MONTH - NG_MONTH
WHERE ABS(EOH_MONTH - (BOH_MONTH + IN_MONTH + BONUS_MONTH - OUT_MONTH - LOSS_MONTH - NG_MONTH)) > 0.01;
```

### 3.3 생산-재고 연결 오류 수정 (우선순위 3)

```sql
-- 1. 불일치 확인
WITH ProdOut AS (
    SELECT YYYYMM, DW_SITE, 도우모델, SUM(OUT_MONTH) AS PROD_OUT
    FROM DOI_PROD_SUBUL
    WHERE 구분 IN ('제품', '반제품')
    GROUP BY YYYYMM, DW_SITE, 도우모델
),
StockIn AS (
    SELECT YYYYMM, SITE, MODEL, SUM(INPUT_PROD) AS STOCK_IN
    FROM DOI_STOCK
    GROUP BY YYYYMM, SITE, MODEL
)
SELECT 
    p.YYYYMM, p.DW_SITE, p.도우모델,
    p.PROD_OUT,
    ISNULL(s.STOCK_IN, 0) AS STOCK_IN,
    p.PROD_OUT - ISNULL(s.STOCK_IN, 0) AS 차이
FROM ProdOut p
LEFT JOIN StockIn s ON p.YYYYMM = s.YYYYMM AND p.DW_SITE = s.SITE AND p.도우모델 = s.MODEL
WHERE ABS(p.PROD_OUT - ISNULL(s.STOCK_IN, 0)) > 0.01;

-- 2. 재고 입고 수정
UPDATE DOI_STOCK
SET INPUT_PROD = (SELECT SUM(OUT_MONTH) FROM DOI_PROD_SUBUL WHERE ...)
WHERE YYYYMM = '202510' AND MODEL = 'MODEL_A';
```

## 🔄 4단계: 월마감 시 검증 실행 (구현 완료 후)

### 4.1 운영자 수동 검증 (월마감 기간)

**검증 화면(C0009001)에서 실행:**

```javascript
// Vue.js 화면 - 검증 실행
async executeValidation() {
    try {
        // 검증 실행
        const result = await this.$axios.post('/api/validation/all', {
            yyyymm: this.selectedMonth,  // 예: '202510'
            site: this.selectedSite      // 'HQ', 'VN', 'ALL'
        });
        
        // 결과 표시
        if (result.data.errorCount > 0) {
            this.$alert(
                `ERROR ${result.data.errorCount}건 발견!\n` +
                `월마감 전 반드시 수정하세요.`,
                'ERROR',
                { type: 'error' }
            );
            this.$router.push('/c0009002');  // 오류 상세 화면
        } else {
            this.$message.success('검증 완료! 모든 데이터가 정상입니다.');
        }
    } catch (error) {
        this.$message.error('검증 실패: ' + error.message);
    }
}
```

### 4.2 월마감 프로세스

```
익월 1일
  └─ [검증 실행] 버튼 클릭 → 오류 확인

익월 3일
  └─ [최종 검증] 버튼 클릭 → ERROR 0건 확인 → 승인
```
