## 🎯 검증 항목

### ST001: 수량수식 검증
- **검증 내용**: `EOH = BOH + INPUT - OUT`
- **대상 테이블**: `DOI_STOCK`
- **오류 표시**: 차이 > 0.01인 경우 빨간색

### ST002: 생산재고연결 검증 ⭐ 핵심
- **검증 내용**: `DOI_FAB_COST.[out] = DOI_STOCK_COST.IN_AMT` (원가항목별)
- **대상 테이블**: `DOI_FAB_COST` ↔ `DOI_STOCK_COST`
- **원가항목**: `expen_sel` 기준 매칭

### ST003: 금액수식 검증
- **검증 내용**: `OUT_AMT = BOH_AMT + IN_AMT - EOH_AMT`
- **대상 테이블**: `DOI_STOCK_COST`
- **오류 표시**: 차이 > 0.01인 경우 빨간색

### ST004: 원가항목 정합성 검증 ⭐ 신규
- **검증 내용**: `expen_sel` NULL 값 존재 여부
- **대상 테이블**: `DOI_FAB_COST`, `DOI_STOCK_BOH`, `DOI_STOCK_COST`
- **오류 유형**: NULL값 존재, 원가항목 누락

### ST005: 음수재고 검증
- **검증 내용**: `BOH < 0` OR `EOH < 0`
- **대상 테이블**: `DOI_STOCK`
- **오류 유형**: BOH 음수, EOH 음수

## 📊 MyBatis 쿼리 특징

### 동적 WHERE 조건
```xml
<where>
  <if test="yyyymm != null and yyyymm != ''">
    AND YYYYMM = #{yyyymm}
  </if>
  <if test="site != null and site != ''">
    AND SITE = #{site}
  </if>
  <if test="model != null and model != ''">
    AND MODEL LIKE '%' + #{model} + '%'
  </if>
</where>
```

### CTE 사용 (ST002 예시)
```sql
WITH FAB_SUMMARY AS (
  SELECT MODEL, 구분, EXPEN_SEL, SUM([OUT]) AS FAB_OUT_AMT
  FROM DOI_FAB_COST
  GROUP BY MODEL, 구분, EXPEN_SEL
),
STOCK_SUMMARY AS (
  SELECT MODEL, 구분, EXPEN_SEL, SUM(IN_AMT) AS STOCK_IN_AMT
  FROM DOI_STOCK_COST
  GROUP BY MODEL, 구분, EXPEN_SEL
)
SELECT ... FROM FAB_SUMMARY FULL OUTER JOIN STOCK_SUMMARY ...
```
