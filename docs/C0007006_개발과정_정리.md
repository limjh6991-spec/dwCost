### 1. Mapper XML 작성

**파일 위치**: `/src/main/resources/mapper/web/c0007000/C0007006.xml`  
**파일 크기**: 77줄  
**네임스페이스**: `com.dowinsys.cost.web.c0007000.mapper.C0007006Mapper`

#### 1.1 밸런스 체크 쿼리

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
  "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.dowinsys.cost.web.c0007000.mapper.C0007006Mapper">

  <!-- 생산수불 밸런스 체크 -->
  <select id="selectBalanceCheck" resultType="CamelMap">
    SELECT 
      YYYYMM,
      DW_MODEL,
      DW_SITE AS SITE,
      MODEL,
      [INCH] AS INCH,
      BOH_MONTH,
      IN_MONTH,
      OUT_MONTH,
      (NG_MONTH + LOSS_MONTH) AS TOTAL_NG,
      EOH_MONTH,
      (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH) AS CALC_EOH,
      (EOH_MONTH - (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH)) AS DIFF_QTY,
      CASE 
        WHEN ABS(EOH_MONTH - (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH)) &lt;= 0.01 
          THEN '정상'
        WHEN ABS(EOH_MONTH - (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH)) &lt;= 10 
          THEN '경고'
        ELSE '오류'
      END AS STATUS,
      CASE 
        WHEN ABS(EOH_MONTH - (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH)) &gt; 10 
          THEN '차이수량이 10을 초과합니다'
        WHEN ABS(EOH_MONTH - (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH)) &gt; 0.01 
          THEN '차이수량이 발생했습니다'
        ELSE '정상'
      END AS REMARK
    FROM DOI_PROD_SUBUL
    WHERE YYYYMM = #{yyyymm}
      AND DW_SITE = #{site}
      <if test="dwModel != null and dwModel != ''">
        AND DW_MODEL LIKE '%' + #{dwModel} + '%'
      </if>
    ORDER BY DW_MODEL, MODEL, [INCH]
  </select>

</mapper>
```

**쿼리 특징**:

1. **밸런스 공식**:
   ```
   기말재고(실제) = 기초재고 + 투입 - 생산 - 불량 - Loss
   EOH_MONTH = BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH
   ```

2. **상태 분류 로직**:
   - 차이 ≤ 0.01: **정상** (부동소수점 오차 허용)
   - 0.01 < 차이 ≤ 10: **경고**
   - 차이 > 10: **오류**

3. **XML 특수문자 처리**:
   - `<=` → `&lt;=` ❌ (잘못된 방법)
   - `<=` → `&lt;` + `=` (별도 처리) ❌
   - **최종**: `&lt;` (less than), `&gt;` (greater than) ✅

4. **INCH 컬럼**: SQL Server 예약어 가능성 → `[INCH]`로 escape

5. **동적 쿼리**: dwModel 파라미터가 있을 때만 LIKE 조건 추가

#### 1.2 요약 정보 쿼리

```xml
<!-- 생산수불 요약 정보 -->
<select id="selectSummary" resultType="CamelMap">
  SELECT 
    COUNT(*) AS TOTAL_COUNT,
    SUM(CASE 
      WHEN ABS(EOH_MONTH - (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH)) &lt;= 0.01 
        THEN 1 ELSE 0 
    END) AS NORMAL_COUNT,
    SUM(CASE 
      WHEN ABS(EOH_MONTH - (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH)) &gt; 10 
        THEN 1 ELSE 0 
    END) AS ERROR_COUNT,
    SUM(CASE 
      WHEN ABS(EOH_MONTH - (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH)) &gt; 0.01 
        AND ABS(EOH_MONTH - (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH)) &lt;= 10 
        THEN 1 ELSE 0 
    END) AS WARNING_COUNT
  FROM DOI_PROD_SUBUL
  WHERE YYYYMM = #{yyyymm}
    AND DW_SITE = #{site}
</select>
```

**쿼리 특징**:
- 집계 쿼리: COUNT, SUM, CASE 활용

### 3. MapperEnum 등록 ⭐ 핵심!

**파일 위치**: `/src/main/java/com/dowinsys/cost/common/enums/MapperEnum.java`

#### 문제 발생
```
Error: No mapper found for menu ID 'c0007006'
```

#### 원인 분석
Generic Search API는 `menuId`를 받아 해당 Mapper 클래스를 찾는 구조인데, `MapperEnum`에 `C0007006`이 등록되어 있지 않았음.

#### 해결 방법

```java
public enum MapperEnum {
    
    // 기존 코드
    C0001004("com.dowinsys.cost.web.c0001000.mapper.C0001004Mapper."),
    C0001007("com.dowinsys.cost.web.c0001000.mapper.C0001007Mapper."),
    C0001009("com.dowinsys.cost.web.c0001000.mapper.C0001009Mapper."),
    
    // 타시스템 I/F
    C0007001("com.dowinsys.cost.web.c0007000.mapper.C0007001Mapper."), // 부서별,계정별 비용
    C0007002("com.dowinsys.cost.web.c0007000.mapper.C0007002Mapper."), // 자재투입정보
    C0007003("com.dowinsys.cost.web.c0007000.mapper.C0007003Mapper."), // 생산정보
    C0007004("com.dowinsys.cost.web.c0007000.mapper.C0007004Mapper."), // 제품정보
    C0007005("com.dowinsys.cost.web.c0007000.mapper.C0007005Mapper."), // 매출 정보
    C0007006("com.dowinsys.cost.web.c0007000.mapper.C0007006Mapper."), // ⭐ 생산수불 자체 체크 추가!
    
    // ... 이하 생략 ...
    
    private final String mapperName;

    MapperEnum(String mapperName) {
        this.mapperName = mapperName;
    }

    public String getMapperName() {
        return this.mapperName;
    }

    /**
     * menuId로 Mapper 클래스 경로 찾기
     */
    public static String getMapperNameByCode(String menuId) {
        for (MapperEnum mapperEnum : values()) {
            if (mapperEnum.name().equalsIgnoreCase(menuId)) {
                return mapperEnum.getMapperName();
            }
        }
        return null; // ← 이게 반환되면 "No mapper found" 에러 발생
    }
}
```

### 문제 3: MyBatis XML 파싱 오류

**에러 메시지**:
```
org.xml.sax.SAXParseException: The content of elements must consist of 
well-formed character data or markup.
```

**발생 시점**: Spring Boot 시작 시 Mapper XML 로드

**원인**: XML에서 `<=`, `>` 같은 특수문자 직접 사용

#### 시도 1: CDATA 섹션 사용 ❌

```xml
<![CDATA[
  WHEN ABS(...) <= 0.01 THEN '정상'
  WHEN ABS(...) <= 10 THEN '경고'
]]>
```

**결과**: 실패 - WHERE 절과 충돌, 동적 쿼리(`<if>`) 사용 불가

#### 시도 2: XML 엔티티 (잘못된 방법) ❌

```xml
WHEN ABS(...) &lt;= 0.01 THEN '정상'
```

**결과**: 실패 - `&lt;=`는 하나의 엔티티가 아님

#### 최종 해결: 올바른 XML 엔티티 사용 ✅

```xml
<!-- <= 대신 < 사용 (0.01 이하 = 0.01 미만) -->
WHEN ABS(...) &lt; 0.01 THEN '정상'

<!-- <= 사용 시 -->
WHEN ABS(...) &lt;= 0.01 THEN '정상'  <!-- 이렇게는 안됨 -->
WHEN ABS(...) &lt; 0.02 THEN '정상'  <!-- 대신 임계값 조정 -->

<!-- > 사용 -->
WHEN ABS(...) &gt; 10 THEN '오류'
```

**XML 엔티티 참조표**:
| 기호 | 엔티티 | 설명 |
|------|--------|------|
| `<` | `&lt;` | less than |
| `>` | `&gt;` | greater than |
| `&` | `&amp;` | ampersand |
| `"` | `&quot;` | quotation mark |
| `'` | `&apos;` | apostrophe |

**배운 점**: XML에서는 특수문자를 엔티티로 변환해야 함

## 데이터베이스 구조

### 테이블 정보

**테이블명**: `DOI_PROD_SUBUL`  
**용도**: 생산수불 데이터 (Production Inventory Ledger)  
**소유자**: 도우제조MES시스템TEST 데이터베이스

### 주요 컬럼

| 컬럼명 | 데이터 타입 | 설명 | 예시 |
|--------|------------|------|------|
| `YYYYMM` | VARCHAR(6) | 기준년월 | 202510 |
| `DW_SITE` | VARCHAR(10) | 사업장 | HQ, VN |
| `DW_MODEL` | VARCHAR(50) | 도우모델 | DOW-001 |
| `MODEL` | VARCHAR(50) | 제품모델 | MODEL-A |
| `INCH` | VARCHAR(10) | 인치 | 10.1 |
| `BOH_MONTH` | DECIMAL(18,2) | 기초재고 (Beginning) | 1000.00 |
| `IN_MONTH` | DECIMAL(18,2) | 당월입고 | 500.00 |
| `OUT_MONTH` | DECIMAL(18,2) | 당월출고 | 450.00 |
| `NG_MONTH` | DECIMAL(18,2) | 당월불량 | 30.00 |
| `LOSS_MONTH` | DECIMAL(18,2) | 당월Loss | 10.00 |
| `EOH_MONTH` | DECIMAL(18,2) | 기말재고 (Ending) | 1010.00 |

### 밸런스 검증 공식

```
기말재고(실제) = 기초재고 + 투입 - 생산 - 불량 - Loss

수식으로 표현:
EOH_MONTH = BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH

차이수량 계산:
DIFF_QTY = EOH_MONTH - (BOH_MONTH + IN_MONTH - OUT_MONTH - NG_MONTH - LOSS_MONTH)
```

### 상태 분류 기준

| 상태 | 조건 | 설명 |
|------|------|------|
| 정상 | `ABS(DIFF_QTY) <= 0.01` | 부동소수점 오차 허용 |
| 경고 | `0.01 < ABS(DIFF_QTY) <= 10` | 소량 차이 발생 |
| 오류 | `ABS(DIFF_QTY) > 10` | 대량 차이 발생 |

### 샘플 데이터

```sql
-- 정상 케이스
YYYYMM: 202510
DW_MODEL: DOW-001
BOH_MONTH: 1000.00
IN_MONTH: 500.00
OUT_MONTH: 450.00
NG_MONTH: 30.00
LOSS_MONTH: 10.00
EOH_MONTH: 1010.00
→ DIFF_QTY = 1010 - (1000 + 500 - 450 - 30 - 10) = 0 (정상)

-- 경고 케이스
EOH_MONTH: 1015.00
→ DIFF_QTY = 5.00 (경고)

-- 오류 케이스
EOH_MONTH: 1050.00
→ DIFF_QTY = 40.00 (오류)
```

**주의사항**:
- `menuId`는 MapperEnum에 등록된 값
- `queryId`는 Mapper XML의 `<select id="...">` 값
- `target`은 배열로 전달 (자동으로 결과 push)

### 5. MyBatis 구조 이해

**3단 구성**:
```
1. XML (SQL 정의)
   ↓
2. Interface (메서드 정의)
   ↓
3. MapperEnum (menuId 매핑)
   ↓
4. Service에서 호출
```

**동작 흐름**:
```
Controller
  ↓
GenericResourceService
  ↓
MapperEnum.getMapperNameByCode(menuId)
  ↓
SqlSession.selectList(mapperId + queryId, params)
  ↓
MyBatis → Mapper Interface → Mapper XML
  ↓
Database
```
