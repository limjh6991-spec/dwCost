## 2. 검증 항목

### 2.1 ST001: 수량 수식 검증 ⭐ 기본

**검증 내용**: `EOH = BOH + INPUT - OUT`

```sql
-- 검증 대상: DOI_STOCK
-- 기대값: 0건
SELECT 
    '수량수식' AS 검증항목,
    MODEL,
    STOCK AS 재고구분,
    BOH, INPUT, OUT, EOH,
    (BOH + INPUT - OUT) AS 계산EOH,
    EOH - (BOH + INPUT - OUT) AS 차이
FROM DOI_STOCK
WHERE YYYYMM = @YYYYMM AND SITE = @SITE
  AND ABS(EOH - (BOH + INPUT - OUT)) > 0.01
ORDER BY MODEL;
```

**오류 원인**:
- 수동 데이터 입력 오류
- 입출고 트랜잭션 누락
- 재고 조정 미반영

---

### 2.2 ST002: 생산-재고 연결 검증 ⭐ 핵심

**검증 내용**: `DOI_FAB_COST.[out] = DOI_STOCK_COST.IN_AMT` (원가항목별)

```sql
-- 검증 대상: DOI_FAB_COST ↔ DOI_STOCK_COST
-- 기대값: 0건
WITH FAB_SUMMARY AS (
    SELECT 
        YYYYMM, SITE, model, 구분, expen_sel,
        SUM([out]) AS FAB_OUT_AMT
    FROM DOI_FAB_COST
    WHERE YYYYMM = @YYYYMM AND SITE = @SITE
    GROUP BY YYYYMM, SITE, model, 구분, expen_sel
),
STOCK_SUMMARY AS (
    SELECT 
        YYYYMM, SITE, MODEL, 구분, expen_sel,
        SUM(IN_AMT) AS STOCK_IN_AMT
    FROM DOI_STOCK_COST
    WHERE YYYYMM = @YYYYMM AND SITE = @SITE
    GROUP BY YYYYMM, SITE, MODEL, 구분, expen_sel
)
SELECT 
    '생산재고연결' AS 검증항목,
    COALESCE(f.model, s.MODEL) AS MODEL,
    COALESCE(f.구분, s.구분) AS 구분,
    COALESCE(f.expen_sel, s.expen_sel) AS 원가항목,
    f.FAB_OUT_AMT AS 생산OUT,
    s.STOCK_IN_AMT AS 재고IN,
    ABS(COALESCE(f.FAB_OUT_AMT, 0) - COALESCE(s.STOCK_IN_AMT, 0)) AS 차이금액,
    CASE 
        WHEN f.FAB_OUT_AMT IS NULL THEN '생산데이터 없음'
        WHEN s.STOCK_IN_AMT IS NULL THEN '재고데이터 없음'
        ELSE '금액 불일치'
    END AS 오류유형
FROM FAB_SUMMARY f
FULL OUTER JOIN STOCK_SUMMARY s
    ON f.YYYYMM = s.YYYYMM 
   AND f.SITE = s.SITE 
   AND f.model = s.MODEL 
   AND f.구분 = s.구분
   AND f.expen_sel = s.expen_sel  -- ⭐ 원가항목별 매칭
WHERE ABS(COALESCE(f.FAB_OUT_AMT, 0) - COALESCE(s.STOCK_IN_AMT, 0)) > 0.01
ORDER BY 차이금액 DESC;
```

**오류 원인**:
- C0003005 가공비 배부 미실행
- expen_sel 불일치 (원가항목 누락)
- 모델 매핑 오류

### 2.3 ST003: 금액 수식 검증 ⭐ 기본

**검증 내용**: `OUT_AMT = BOH_AMT + IN_AMT - EOH_AMT`

```sql
-- 검증 대상: DOI_STOCK_COST
-- 기대값: 0건
SELECT 
    '금액수식' AS 검증항목,
    MODEL,
    구분,
    expen_sel AS 원가항목,
    BOH_AMT, IN_AMT, EOH_AMT, OUT_AMT,
    (BOH_AMT + IN_AMT - EOH_AMT) AS 계산OUT_AMT,
    OUT_AMT - (BOH_AMT + IN_AMT - EOH_AMT) AS 차이
FROM DOI_STOCK_COST
WHERE YYYYMM = @YYYYMM AND SITE = @SITE
  AND ABS(OUT_AMT - (BOH_AMT + IN_AMT - EOH_AMT)) > 0.01
ORDER BY ABS(차이) DESC;
```

**오류 원인**:
- 총평균법 계산 오류
- 반올림 오차 누적
- NULL 처리 미흡

### 2.4 ST004: 원가항목 정합성 검증 ⭐ 신규 (expen_sel 추가로 필수)

**검증 내용**: 
- expen_sel NULL 값 존재 여부
- DOI_FAB_COST, DOI_STOCK_BOH, DOI_STOCK_COST 간 expen_sel 일치 여부

```sql
-- 검증 1: NULL 체크
SELECT 
    '원가항목NULL' AS 검증항목,
    '테이블명' AS 테이블,
    COUNT(*) AS NULL건수
FROM (
    SELECT 'DOI_FAB_COST' AS 테이블명, COUNT(*) AS CNT
    FROM DOI_FAB_COST
    WHERE YYYYMM = @YYYYMM AND SITE = @SITE AND expen_sel IS NULL
    
    UNION ALL
    
    SELECT 'DOI_STOCK_BOH', COUNT(*)
    FROM DOI_STOCK_BOH
    WHERE YYYYMM = @YYYYMM AND SITE = @SITE AND expen_sel IS NULL
    
    UNION ALL
    
    SELECT 'DOI_STOCK_COST', COUNT(*)
    FROM DOI_STOCK_COST
    WHERE YYYYMM = @YYYYMM AND SITE = @SITE AND expen_sel IS NULL
) x
WHERE CNT > 0;

-- 검증 2: 원가항목 불일치
WITH EXPECTED_EXPEN AS (
    SELECT DISTINCT expen_sel
    FROM DOI_FAB_COST
    WHERE YYYYMM = @YYYYMM AND SITE = @SITE
      AND expen_sel IS NOT NULL
)
SELECT 
    '원가항목불일치' AS 검증항목,
    s.MODEL,
    s.구분,
    e.expen_sel AS 기대원가항목,
    s.expen_sel AS 실제원가항목,
    '누락' AS 오류유형
FROM EXPECTED_EXPEN e
CROSS JOIN (
    SELECT DISTINCT MODEL, 구분 
    FROM DOI_STOCK_COST 
    WHERE YYYYMM = @YYYYMM AND SITE = @SITE
) m
LEFT JOIN DOI_STOCK_COST s
    ON s.YYYYMM = @YYYYMM 
   AND s.SITE = @SITE
   AND s.MODEL = m.MODEL
   AND s.구분 = m.구분
   AND s.expen_sel = e.expen_sel
WHERE s.expen_sel IS NULL
ORDER BY MODEL, 기대원가항목;
```

**오류 원인**:
- DOI_FAB_COST에 expen_sel 누락
- JOIN 조건 불일치
- 데이터 마이그레이션 오류

### 2.5 ST005: 음수 재고 검증 ⭐ 운영

**검증 내용**: BOH < 0 또는 EOH < 0 존재 여부

```sql
-- 검증 대상: DOI_STOCK, DOI_STOCK_COST
-- 기대값: 0건
SELECT 
    '음수재고' AS 검증항목,
    MODEL,
    STOCK AS 재고구분,
    BOH AS 기초재고,
    EOH AS 기말재고,
    CASE 
        WHEN BOH < 0 THEN 'BOH 음수'
        WHEN EOH < 0 THEN 'EOH 음수'
        ELSE '기타'
    END AS 오류유형
FROM DOI_STOCK
WHERE YYYYMM = @YYYYMM AND SITE = @SITE
  AND (BOH < 0 OR EOH < 0)
ORDER BY MODEL;
```

**오류 원인**:
- 출고가 입고보다 먼저 발생
- 재고 초기화 오류
- 이월 재고 미반영

### 4.2 프로시저 제안 (선택사항)

```sql
CREATE PROCEDURE UP_DOI_STOCK_VALIDATION
(
    @YYYYMM VARCHAR(6),
    @SITE VARCHAR(2),
    @ValidationCode VARCHAR(10) = NULL,  -- NULL이면 전체 검증
    @R_Message NVARCHAR(MAX) OUTPUT
)
AS
BEGIN
    SET NOCOUNT ON;
    
    -- 임시 테이블 생성
    CREATE TABLE #ValidationResult (
        seq INT IDENTITY(1,1),
        validCode VARCHAR(10),
        validName NVARCHAR(50),
        model VARCHAR(50),
        gubun VARCHAR(20),
        expenSel VARCHAR(20),
        expenSelName NVARCHAR(100),
        errorType NVARCHAR(100),
        diffAmt DECIMAL(18,2),
        fabOut DECIMAL(18,2),
        stockIn DECIMAL(18,2),
        detail NVARCHAR(MAX)
    );
    
    -- ST001: 수량 수식 검증
    IF @ValidationCode IS NULL OR @ValidationCode = 'ST001'
    BEGIN
        INSERT INTO #ValidationResult (validCode, validName, model, gubun, errorType, diffAmt, detail)
        SELECT 
            'ST001',
            '수량수식',
            MODEL,
            STOCK,
            '수량 불일치',
            EOH - (BOH + INPUT - OUT),
            'BOH:' + CAST(BOH AS VARCHAR) + ', IN:' + CAST(INPUT AS VARCHAR) + 
            ', OUT:' + CAST(OUT AS VARCHAR) + ', EOH:' + CAST(EOH AS VARCHAR)
        FROM DOI_STOCK
        WHERE YYYYMM = @YYYYMM AND SITE = @SITE
          AND ABS(EOH - (BOH + INPUT - OUT)) > 0.01;
    END
    
    -- ST002: 생산-재고 연결 검증 (expen_sel 포함)
    IF @ValidationCode IS NULL OR @ValidationCode = 'ST002'
    BEGIN
        WITH FAB_SUMMARY AS (
            SELECT model, 구분, expen_sel, SUM([out]) AS FAB_OUT_AMT
            FROM DOI_FAB_COST
            WHERE YYYYMM = @YYYYMM AND SITE = @SITE
            GROUP BY model, 구분, expen_sel
        ),
        STOCK_SUMMARY AS (
            SELECT MODEL, 구분, expen_sel, SUM(IN_AMT) AS STOCK_IN_AMT
            FROM DOI_STOCK_COST
            WHERE YYYYMM = @YYYYMM AND SITE = @SITE
            GROUP BY MODEL, 구분, expen_sel
        )
        INSERT INTO #ValidationResult (
            validCode, validName, model, gubun, expenSel, 
            errorType, diffAmt, fabOut, stockIn
        )
        SELECT 
            'ST002',
            '생산재고연결',
            COALESCE(f.model, s.MODEL),
            COALESCE(f.구분, s.구분),
            COALESCE(f.expen_sel, s.expen_sel),
            CASE 
                WHEN f.FAB_OUT_AMT IS NULL THEN '생산데이터 없음'
                WHEN s.STOCK_IN_AMT IS NULL THEN '재고데이터 없음'
                ELSE '금액 불일치'
            END,
            ABS(COALESCE(f.FAB_OUT_AMT, 0) - COALESCE(s.STOCK_IN_AMT, 0)),
            COALESCE(f.FAB_OUT_AMT, 0),
            COALESCE(s.STOCK_IN_AMT, 0)
        FROM FAB_SUMMARY f
        FULL OUTER JOIN STOCK_SUMMARY s
            ON f.model = s.MODEL 
           AND f.구분 = s.구분
           AND f.expen_sel = s.expen_sel
        WHERE ABS(COALESCE(f.FAB_OUT_AMT, 0) - COALESCE(s.STOCK_IN_AMT, 0)) > 0.01;
    END
    
    -- ST003, ST004, ST005도 유사하게 추가...
    
    -- 결과 반환
    SELECT * FROM #ValidationResult
    ORDER BY validCode, diffAmt DESC;
    
    -- 요약 정보
    SET @R_Message = '검증 완료: 총 ' + CAST((SELECT COUNT(*) FROM #ValidationResult) AS VARCHAR) + '건 오류 발견';
    
    DROP TABLE #ValidationResult;
END;
```

### 5.1 MyBatis Mapper (C0003009Check.xml)

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" 
  "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.dowinsys.cost.mapper.C0003009CheckMapper">

  <!-- 검증 요약 조회 -->
  <select id="getValidationSummary" resultType="ValidationSummaryVO">
    SELECT 
      validCode,
      COUNT(*) AS total,
      SUM(CASE WHEN errorCnt = 0 THEN 1 ELSE 0 END) AS success,
      SUM(CASE WHEN errorCnt > 0 THEN 1 ELSE 0 END) AS error
    FROM (
      -- ST001: 수량수식
      SELECT 'ST001' AS validCode, 
             COUNT(*) AS errorCnt
      FROM DOI_STOCK
      WHERE YYYYMM = #{yyyymm} AND SITE = #{site}
        AND ABS(EOH - (BOH + INPUT - OUT)) > 0.01
      
      UNION ALL
      
      -- ST002: 생산재고연결 (원가항목별)
      SELECT 'ST002', COUNT(*)
      FROM (
        SELECT f.model, f.구분, f.expen_sel
        FROM DOI_FAB_COST f
        FULL OUTER JOIN DOI_STOCK_COST s
          ON f.YYYYMM = s.YYYYMM 
         AND f.SITE = s.SITE
         AND f.model = s.MODEL 
         AND f.구분 = s.구분
         AND f.expen_sel = s.expen_sel
        WHERE f.YYYYMM = #{yyyymm} AND f.SITE = #{site}
          AND ABS(COALESCE(f.[out], 0) - COALESCE(s.IN_AMT, 0)) > 0.01
      ) x
      
      -- ST003, ST004, ST005도 추가...
    ) summary
    GROUP BY validCode
  </select>

  <!-- 상세 오류 조회 -->
  <select id="getValidationDetail" resultType="ValidationDetailVO">
    <if test="validCode == 'ST002'">
      WITH FAB_SUMMARY AS (
        SELECT model, 구분, expen_sel, SUM([out]) AS FAB_OUT_AMT
        FROM DOI_FAB_COST
        WHERE YYYYMM = #{yyyymm} AND SITE = #{site}
        GROUP BY model, 구분, expen_sel
      ),
      STOCK_SUMMARY AS (
        SELECT MODEL, 구분, expen_sel, SUM(IN_AMT) AS STOCK_IN_AMT
        FROM DOI_STOCK_COST
        WHERE YYYYMM = #{yyyymm} AND SITE = #{site}
        GROUP BY MODEL, 구분, expen_sel
      )
      SELECT 
        'ST002' AS validCode,
        COALESCE(f.model, s.MODEL) AS model,
        COALESCE(f.구분, s.구분) AS gubun,
        COALESCE(f.expen_sel, s.expen_sel) AS expenSel,
        CASE 
          WHEN f.FAB_OUT_AMT IS NULL THEN '생산데이터 없음'
          WHEN s.STOCK_IN_AMT IS NULL THEN '재고데이터 없음'
          ELSE '금액 불일치'
        END AS errorType,
        ABS(COALESCE(f.FAB_OUT_AMT, 0) - COALESCE(s.STOCK_IN_AMT, 0)) AS diffAmt,
        COALESCE(f.FAB_OUT_AMT, 0) AS fabOut,
        COALESCE(s.STOCK_IN_AMT, 0) AS stockIn
      FROM FAB_SUMMARY f
      FULL OUTER JOIN STOCK_SUMMARY s
        ON f.model = s.MODEL 
       AND f.구분 = s.구분
       AND f.expen_sel = s.expen_sel
      WHERE ABS(COALESCE(f.FAB_OUT_AMT, 0) - COALESCE(s.STOCK_IN_AMT, 0)) > 0.01
      ORDER BY diffAmt DESC
    </if>
  </select>

</mapper>
```

```java
    private String model;
    private String gubun;
    private String expenSel;        // ⭐ 신규
    private String expenSelName;    // ⭐ 신규
    private String errorType;
    private BigDecimal diffAmt;
    private BigDecimal fabOut;
    private BigDecimal stockIn;
}

// Service
@Service
public class C0003009CheckService {
    
    @Autowired
    private C0003009CheckMapper mapper;
    
    public ValidationResultDTO executeValidation(String yyyymm, String site) {
        // 요약 정보 조회
        List<ValidationSummaryVO> summary = mapper.getValidationSummary(yyyymm, site);
        
        // 상세 오류 조회 (모든 검증 항목)
        List<ValidationDetailVO> details = new ArrayList<>();
        for (ValidationSummaryVO item : summary) {
            if (item.getError() > 0) {
                details.addAll(mapper.getValidationDetail(yyyymm, site, item.getValidCode()));
            }
        }
        
        return ValidationResultDTO.builder()
            .summary(summary)
            .details(details)
            .totalError(details.size())
            .build();
    }
}

// Controller
@RestController
@RequestMapping("/api/c0003009/check")
public class C0003009CheckController {
    
    @Autowired
    private C0003009CheckService service;
    
    @PostMapping("/validate")
    public ResponseEntity<ValidationResultDTO> validate(@RequestBody ValidationParam param) {
        ValidationResultDTO result = service.executeValidation(
            param.getYyyymm(),

    }
```
