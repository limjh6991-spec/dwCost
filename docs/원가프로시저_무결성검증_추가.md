# 원가 프로시저 데이터 무결성 검증 로직 추가 완료

## 수정 내용

모든 원가 프로시저에 **데이터 무결성 검증 로직**을 추가했습니다.

### 1. Gen_EXPEN_MATL (경비집계)

**검증 항목:**
- 소스 테이블(DOI_ACCT_EXPEN) 금액 합계
- 타겟 테이블(DOI_EXPEN_MATL) 금액 합계
- 금액 차이 1원 이상 시 경고

**누락 원인 확인:**
- 누락된 부서 목록 (DOI_DEPT 마스터 미등록)
- 누락된 계정 목록
- 조인 실패 원인 (DOI_DEPT 마스터에 없는 부서 개수)

**메시지 예시:**
```
[CHECK] - 소스금액: 123,456,789원, 타겟금액: 123,456,789원, 차이: 0원
[CHECK] - 경비집계 데이터 무결성 검증 완료 (일치)
```

---

### 2. UP_DOI_FAB_COST (공비배부)

**검증 항목:**
- 소스 테이블(DOI_EXPEN_MATL) 금액 합계
- 타겟 테이블(DOI_FAB_COST - 가공비만) 금액 합계
- 금액 차이 체크

**메시지 예시:**
```
[CHECK] - 소스금액(경비집계): 123,456,789원, 타겟금액(가공비배부): 123,456,789원, 차이: 0원
[CHECK] - 가공비배부 데이터 무결성 검증 완료 (일치)
```

---

### 3. UP_DOI_MAT_AMT (재료비집계)

**검증 항목:**
- 소스 테이블(DOI_MATERIAL_RESC) 금액/수량 합계
- 타겟 테이블(DOI_MAT_AMT) 금액/수량 합계
- 금액 및 수량 차이 체크

**누락 원인 확인:**
- 누락된 자재 목록 (상위 10개)
- 원인: BOM 마스터 미등록 또는 in_amt <= 0 필터링

**메시지 예시:**
```
[CHECK] - 소스금액: 50,000,000원 (1,234.56개), 타겟금액: 50,000,000원 (1,234.56개)
[CHECK] - 금액차이: 0원, 수량차이: 0.00개
[CHECK] - 재료비집계 데이터 무결성 검증 완료 (일치)
```

---

### 4. UP_DOI_MAT_COST (재료비배부)

**검증 항목:**
- 소스 테이블(DOI_MAT_AMT) 금액 합계
- 타겟 테이블(DOI_MAT_COST) 배부금액 합계
- 금액 차이 체크

**누락 원인 확인:**
- BOM 마스터 미등록 자재 개수
- 배부되지 않은 모델 개수

**메시지 예시:**
```
[CHECK] - 소스금액(재료비집계): 50,000,000원, 타겟금액(재료비배부): 50,000,000원, 차이: 0원
[CHECK] - 재료비배부 데이터 무결성 검증 완료 (일치)
```

---

### 5. UP_DOI_COST_MAT (재공평가)

**검증 항목:**
- 소스 테이블(DOI_MAT_COST) 배부금액 합계
- 타겟 테이블(DOI_FAB_COST - 재료비만) 금액 합계
- 금액 차이 체크

**메시지 예시:**
```
[CHECK] - 소스금액(재료비배부): 50,000,000원, 타겟금액(재공평가): 50,000,000원, 차이: 0원
[CHECK] - 재공평가(재료비) 데이터 무결성 검증 완료 (일치)
```

---

### 6. UP_DOI_STOCK_COST (제품수불)

**검증 항목:**
- 소스 테이블(DOI_STOCK_BOH) 기초재고 금액
- 소스 테이블(DOI_FAB_COST) 투입금액
- 타겟 테이블(DOI_STOCK_COST) BOH, IN, EOH, OUT 금액
- 밸런스 체크: BOH + IN = EOH + OUT

**메시지 예시:**
```
[CHECK] - 기초재고: 소스=10,000,000원, 타겟=10,000,000원, 차이=0원
[CHECK] - 투입금액: 소스=100,000,000원, 타겟=100,000,000원, 차이=0원
[CHECK] - 기말재고: 15,000,000원, 출고금액: 95,000,000원
[CHECK] - 밸런스 체크(BOH+IN=EOH+OUT): 0원
[CHECK] - 제품수불 데이터 무결성 검증 완료 (일치)
```

---

## 적용 방법

수정된 프로시저 파일들을 SQL Server에 업데이트해야 합니다:

1. **Gen_EXPEN_MATL.sql** - 경비집계
2. **UP_DOI_FAB_COST** - 공비배부
3. **UP_DOI_MAT_AMT** - 재료비집계
4. **UP_DOI_MAT_COST** - 재료비배부
5. **UP_DOI_COST_MAT** - 재공평가
6. **UP_DOI_STOCK_COST** - 제품수불

각 파일을 DBeaver 또는 SSMS에서 실행하여 프로시저를 업데이트하세요.

---

## 검증 기준

- **금액 차이 1원 이하**: 정상 (반올림 오차 허용)
- **금액 차이 1원 초과**: 경고 발생, 누락 원인 분석 메시지 출력

---

## 기대 효과

1. ✅ **데이터 누락 즉시 감지**: 프로시저 실행 시 자동으로 데이터 무결성 확인
2. ✅ **누락 원인 자동 분석**: 어떤 부서/계정/자재가 누락되었는지 자동 확인
3. ✅ **밸런스 체크**: 제품수불의 경우 BOH+IN=EOH+OUT 자동 검증
4. ✅ **운영 안정성 향상**: 데이터 품질 문제를 사전에 방지
