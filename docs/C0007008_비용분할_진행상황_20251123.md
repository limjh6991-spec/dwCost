## 📋 작업 개요

### 목표
부서별 비용(DOI_DEPT_COST)을 다음 4개 테이블로 분할 배부:
1. **DOI_MFG_EXP**: 제조경비 (Manufacturing Expenses)
2. **DOI_SGA**: 판매관리비 (Sales, General & Administrative)
3. **DOI_RND**: 연구개발비 (Research & Development)
4. **DOI_CASET**: 카세트팀 비용 (Cassette Team Cost)

### 기준 월
- 202507, 202510 (202508, 202509는 데이터 없음)

### 1. DOI_DEPT 테이블 컬럼 추가 (2025-11-23)

#### 추가 컬럼
```sql
ALTER TABLE DOI_DEPT ADD RND_YN VARCHAR(1) NULL;           -- 연구개발 여부
ALTER TABLE DOI_DEPT ADD COST_DIST NVARCHAR(50) NULL;      -- 원가 구분
ALTER TABLE DOI_DEPT ADD COST_DIST_RATE DECIMAL(5,2) NULL; -- 배부 비율
```

#### 데이터 매핑 (202510 기준)
- **RND_YN**: 
  - 'Y': 18개 부서 (연구개발 부서)
  - 'N': 62개 부서 (일반 부서)
  - NULL: 6개 부서

- **COST_DIST**:
  - '원가': 32개 부서
  - '총원가': 28개 부서
  - '총원가-연구개발': 20개 부서

- **COST_DIST_RATE**:
  - 100.00: 25개 부서 (단일 배부)
  - 90.00: 3개 부서 (원가/총원가 혼합)
  - 10.00: 13개 부서 (원가/총원가 혼합)

#### 백업
- 백업 테이블: `BCK_DOI_DEPT` (86건)

### 2. 검증 테이블 생성 (2025-11-23)

4개 테이블 모두 동일한 구조:

```sql
CREATE TABLE DOI_MFG_EXP (
    YYYYMM VARCHAR(6) NOT NULL,           -- 기준년월
    SEL_CODE VARCHAR(6) NOT NULL,         -- 선택코드
    SITE VARCHAR(4) NOT NULL,             -- 사이트
    FROM_DEPT_CODE VARCHAR(10) NOT NULL,  -- 부서코드
    FROM_DEPT_NAME NVARCHAR(100) NULL,    -- 부서명
    FROM_ACCT_CODE VARCHAR(10) NOT NULL,  -- 계정코드
    FROM_ACCT_NAME NVARCHAR(100) NULL,    -- 계정과목
    EXPEN_SEL NVARCHAR(8) NULL,           -- 경비구분
    EXPEN_SEL_NAME NVARCHAR(60) NULL,     -- 경비구분명
    ACCT_AMT BIGINT NULL,                 -- 차변금액
    PRIMARY KEY (YYYYMM, SEL_CODE, SITE, FROM_DEPT_CODE, FROM_ACCT_CODE)
);
```

#### 테이블별 설명
- **DOI_MFG_EXP**: 제조경비 검증 테이블
- **DOI_SGA**: 판매관리비 검증 테이블
- **DOI_RND**: 연구개발비 검증 테이블
- **DOI_CASET**: 카세트팀 비용 검증 테이블

#### 주요 수정사항

1. **원본 금액 계산 수정**
   ```sql
   -- 수정 전: DOI_DEPT 조인 (중복 카운트 발생)
   SELECT @원본금액 = SUM(dc.차변금액)
   FROM DOI_DEPT_COST dc
   INNER JOIN DOI_DEPT d ON ...
   
   -- 수정 후: 직접 합계 (중복 없음)
   SELECT @원본금액 = SUM(차변금액)
   FROM DOI_DEPT_COST
   WHERE YYYYMM = @YYYYMM
       AND SEL_CODE = @SEL_CODE
       AND SITE = @SITE
       AND 차변금액 > 0
   ```

2. **COST_DIST NULL 처리 추가**
   ```sql
   -- 제조경비: COST_DIST가 NULL이면 '원가'로 처리
   AND (d.COST_DIST = N'원가' OR d.COST_DIST IS NULL)
   ```

3. **총원가 변형 패턴 지원**
   ```sql
   -- 판관비: '총원가-연구개발' 등 변형 패턴 포함
   AND (d.COST_DIST = N'총원가' OR d.COST_DIST LIKE N'총원가%')
   ```

4. **카세트팀 SITE 처리 개선**

   ```sql
   -- 수정 전: dc.SITE = @SITE 조건으로 VN 데이터 누락
   -- 수정 후: 조건 제거, HQ/VN 모두 정상 배부
   ```

5. **COLLATE 제거**: 단순 비교로 변경

7. **연구개발비 100% 배부**

### 5. 배부 로직 (최종 확정)

#### 제조경비 (DOI_MFG_EXP)
- **대상**: COST_DIST = '원가' OR NULL
- **제외**: 카세트팀 (DEPT='400','448')
- **금액**: `차변금액 × COST_DIST_RATE / 100`

#### 판관비 (DOI_SGA)
- **대상**: COST_DIST LIKE '총원가%' AND RND_YN='N'
- **금액**: `차변금액 × COST_DIST_RATE / 100`

#### 연구개발비 (DOI_RND)
- **대상**: RND_YN='Y'
- **금액**: `차변금액` (100%)
- **추가**: COST_TYPE (인건비 → 판관비, 기타 → 제조비용)

#### 카세트팀 (DOI_CASET)
- **대상**: DEPT IN ('400','448')
- **HQ**: `차변금액 × UTG 비율`
- **VN**: `차변금액 × VINA_CST 비율`

## 🐛 주요 이슈 및 해결

### Issue 1: 원본 금액 중복 카운트
**문제**: DOI_DEPT 조인 시 같은 DEPT_NAME에 여러 DEPT 매핑이 있어 중복 집계
- 예: '전사' → DEPT=001('원가'), DEPT=001('총원가') 2개 행

**해결**: 원본 금액 계산 시 DOI_DEPT_COST 직접 합계 사용

### Issue 2: COST_DIST NULL 처리
**문제**: '판매공통', '제조공통' 등 일부 부서의 COST_DIST가 NULL → 배부 제외됨

**해결**: 제조경비 조건에 `OR d.COST_DIST IS NULL` 추가

### Issue 3: 차변금액 > 0 조건
**문제**: 검증 쿼리와 프로시저의 원본 금액 계산이 불일치

**해결**: 원본 금액 계산에 `AND 차변금액 > 0` 조건 추가

### Issue 4: 카세트팀 VN 데이터 누락
**문제**: `dc.SITE = @SITE` 조건으로 인해 VN INSERT가 실행되지 않음

**해결**: 카세트팀 INSERT에서 SITE 필터 제거
