프로시저 로직이 업데이트되어 **원가항목별 세분화** 기능이 추가되었습니다.

**영향:**

DOI_STOCK {

## 3. UP_DOI_STOCK_BOH 프로시저 상세

### 3.1 Phase 1: 제품 BOH 금액 집계

**목표:** DOI_STOCK_BOH 테이블에 기초재고 금액 산출

```sql
-- Step 1: 모델별 OUT 단가 추출
WITH MODEL_OUT단가 AS (
    SELECT
        a.YYYYMM, a.SITE, a.구분, a.model,
        b.inch,
        a.expen_sel,                      -- ⭐ 원가항목 코드 추가
        MAX(a.expen_sel명) AS expen_sel명, -- ⭐ 원가항목명 추가
        SUM(a.out_단가) AS out_단가,      -- ⭐ 핵심: 가공완료 후 단가
        SUM(a.unit_cost) AS unit_cost,
        AVG(SUM(a.out_단가)) OVER (PARTITION BY inch) AS avg_cost,
        ROW_NUMBER() OVER (PARTITION BY inch, a.model, a.expen_sel ORDER BY a.model, a.expen_sel) AS rn
    FROM DOI_FAB_COST a
    LEFT JOIN DOI_MODEL_MAST b 
        ON a.model = b.model 
       AND a.yyyymm = b.yyyymm 
       AND a.site = b.site
    WHERE a.YYYYMM = @YYYYMM AND a.SITE = @SITE
    GROUP BY a.YYYYMM, a.SITE, a.구분, b.inch, a.model, a.expen_sel
    HAVING SUM(a.out_단가) > 0
)

-- Step 2: BOH 금액 산출 (3가지 매칭 방법)
INSERT INTO DOI_STOCK_BOH

-- 방법 1: 모델 + 구분(양산/개발) 완전 매칭 ⭐ 우선순위 1
SELECT
    a.YYYYMM, a.SITE, a.sel_code,
    CASE WHEN a.stock='개발 창고' THEN '개발' 
         WHEN a.stock='완제품 창고' THEN '양산' 
         ELSE '기타' END AS 구분,
    a.model, a.model_type,
    b.expen_sel,                              -- ⭐ 원가항목 코드
    b.expen_sel명,                            -- ⭐ 원가항목명
    a.stock,
    ROUND(a.boh * b.out_단가, 0) AS boh_amt,  -- 💰 BOH 금액 = 수량 × 단가
    'model_1' AS 조건,
    b.inch,
    a.boh,
    b.out_단가
FROM DOI_STOCK a
INNER JOIN MODEL_OUT단가 b 
    ON a.model = b.model 
   AND b.구분 = CASE WHEN a.stock='개발 창고' THEN '개발' 
                    WHEN a.stock='완제품 창고' THEN '양산' 
                    ELSE '기타' END
WHERE a.YYYYMM = @YYYYMM AND a.SITE = @SITE

UNION

-- 방법 2: 모델만 매칭 (구분 무시) ⭐ 우선순위 2
SELECT
    a.YYYYMM, a.SITE, a.sel_code,
    CASE WHEN a.stock='개발 창고' THEN '개발' 
         WHEN a.stock='완제품 창고' THEN '양산' 
         ELSE '기타' END AS 구분,
    a.model, a.model_type,
    b.expen_sel,                              -- ⭐ 원가항목 코드
    b.expen_sel명,                            -- ⭐ 원가항목명
    a.stock,
    ROUND(a.boh * b.out_단가, 0) AS boh_amt,
    'model_2' AS 조건,
    b.inch,
    a.boh,
    b.out_단가
FROM DOI_STOCK a
INNER JOIN MODEL_OUT단가 b ON a.model = b.model
WHERE a.YYYYMM = @YYYYMM AND a.SITE = @SITE
  AND NOT EXISTS (
    SELECT 1 FROM MODEL_OUT단가 c
    WHERE b.model = c.model 
      AND c.구분 = CASE WHEN a.stock='개발 창고' THEN '개발' 
                       WHEN a.stock='완제품 창고' THEN '양산' 
                       ELSE '기타' END
)

UNION

-- 방법 3: inch 평균단가 사용 (모델 미매칭 시) ⭐ 우선순위 3
SELECT
    a.YYYYMM, a.SITE, a.sel_code,
    CASE WHEN a.stock='개발 창고' THEN '개발' 
         WHEN a.stock='완제품 창고' THEN '양산' 
         ELSE '기타' END AS 구분,
    a.model, a.model_type,
    c.expen_sel,                              -- ⭐ 원가항목 코드
    c.expen_sel명,                            -- ⭐ 원가항목명
    a.stock,
    ROUND(a.boh * COALESCE(c.avg_cost, 0), 0) AS boh_amt,
    'inch' AS 조건,
    b.inch,
    a.boh,
    COALESCE(c.avg_cost, 0) AS out_단가
FROM DOI_STOCK a
LEFT JOIN DOI_MODEL_MAST b ON a.model = b.model
LEFT JOIN MODEL_OUT단가 c 
    ON c.inch = CASE WHEN b.inch IN (7.99, 8.01) THEN 8
                     WHEN b.inch = 7.61 THEN 7.72 
                     WHEN b.inch IN (6.86, 7.55) THEN 7.56 
                     ELSE b.inch END
   AND c.rn = 1
WHERE a.YYYYMM = @YYYYMM AND a.SITE = @SITE
  AND NOT EXISTS (
    SELECT 1 FROM MODEL_OUT단가 c WHERE a.model = c.model
)
ORDER BY a.model;
```

**UP_DOI_STOCK_COST 프로시저 로직:**

```sql
-- 목표: DOI_STOCK_COST 테이블에 BOH/IN/OUT/EOH 금액 산출

-- Step 1: 가공비 배부에서 당월 생산 입고 금액 (IN_AMT)
WITH MODEL_IN_AMT AS (
    SELECT 
        YYYYMM, site, model, 구분, 
        SUM([out]) AS IN_AMT    -- ⭐ 가공비 OUT = 재고 IN
    FROM DOI_FAB_COST
    WHERE yyyymm = @YYYYMM AND site = @SITE
    GROUP BY yyyymm, site, model, 구분
),

-- Step 2: Phase 1에서 산출한 BOH 금액
MODEL_BOH_AMT AS (
    SELECT 
        YYYYMM, site, model, 구분, expen_sel,  -- ⭐ expen_sel 추가
        SUM(BOH_AMT) AS BOH_AMT
    FROM DOI_STOCK_BOH
    WHERE yyyymm = @YYYYMM AND site = @SITE
    GROUP BY yyyymm, site, model, 구분, expen_sel  -- ⭐ GROUP BY에 추가
),

-- Step 3: 수량 집계 (재고구분별 합계)
MODEL_STOCK AS (
    SELECT
        YYYYMM, SITE, SEL_CODE,
        CASE WHEN stock='개발 창고' THEN '개발' 
             WHEN stock='완제품 창고' THEN '양산' 
             ELSE '기타' END AS 구분,
        MODEL,
        SUM(BOH) AS BOH,
        SUM(INPUT) AS INPUT,
        SUM(OUT) AS OUT,
        SUM(EOH) AS EOH
    FROM DOI_STOCK
    WHERE YYYYMM = @YYYYMM AND SITE = @SITE
    GROUP BY YYYYMM, SITE, SEL_CODE, MODEL, stock
),

-- Step 4: EOH 금액 계산 (총평균법)
MODEL_EOH_AMT AS (
    SELECT
        a.YYYYMM, a.SITE, a.SEL_CODE, a.구분, a.MODEL,
        a.BOH, a.INPUT, a.OUT, a.EOH,
        b.BOH_AMT,
        COALESCE(c.IN_AMT, 0) AS IN_AMT,
        
        -- 💰 EOH 금액 계산 로직 (⭐ 핵심 - 총평균법)
        ROUND(
            CASE 
                WHEN a.out = 0 
                    -- 출고가 없으면: BOH + IN 전체가 재고
                    THEN COALESCE(b.boh_amt, 0) + COALESCE(c.IN_AMT, 0)
                ELSE 
                    -- 출고가 있으면: 총평균법 적용
                    -- 평균단가 = (BOH금액 + IN금액) / (BOH수량 + IN수량)
                    -- EOH금액 = 평균단가 × EOH수량
                    (COALESCE(b.boh_amt, 0) + COALESCE(c.IN_AMT, 0)) 
                    / NULLIF((a.eoh + a.out), 0) * a.eoh
            END, 0
        ) AS EOH_AMT
    FROM MODEL_STOCK a
    LEFT JOIN MODEL_BOH_AMT b 
        ON a.yyyymm = b.yyyymm 
       AND a.site = b.site 
       AND a.model = b.model 
       AND b.구분 = a.구분
    LEFT JOIN MODEL_IN_AMT c  
        ON a.yyyymm = c.yyyymm 
       AND a.site = c.site 
       AND a.model = c.model 
       AND c.구분 = a.구분
)

-- Step 5: 최종 INSERT
INSERT INTO DOI_STOCK_COST
SELECT *,
    -- 💰 OUT 금액 = BOH + IN - EOH (총평균법 차감)
    ROUND(
        CASE WHEN eoh = 0 
            THEN (boh_amt + IN_AMT) 
            ELSE (boh_amt + IN_AMT - eoh_amt) 
        END, 0
    ) AS OUT_AMT
FROM MODEL_EOH_AMT
ORDER BY MODEL, 4;
```

### 3.3 Phase 3: 판매관리비 배부

**목표:** DOI_SMCE_COST 테이블에 판매관리비 배부

```sql
-- Step 1: 매출 비율 계산
WITH sale_rate AS (
    SELECT
        yyyymm,
        품명 AS model,
        SUM(원화판매금액) AS SALE_AMT,
        SUM(SUM(원화판매금액)) OVER() AS tot_amt,
        CAST(SUM(원화판매금액) AS INT) 
        / NULLIF(SUM(SUM(원화판매금액)) OVER(), 0) AS dist_rate
    FROM DOI_SALE_RESC
    WHERE yyyymm = @YYYYMM AND site = @SITE
    GROUP BY yyyymm, 품명
),

-- Step 2: 판매관리비 항목 집계
sale_cntr AS (
    SELECT 
        YYYYMM, SITE, SUB_NAME, ITEM_NAME, EXPEN_SEL, EXPEN_SEL명,
        SUM(ACCT_AMT) AS TOT_ACCT,
        SUM(SUM(ACCT_AMT)) OVER() AS TOT_SMCE
    FROM DOI_ACCT_EXPEN
    WHERE yyyymm = @YYYYMM 
      AND site = @SITE
      AND acct_class = 'CC'  -- 판매관리비만
    GROUP BY YYYYMM, SITE, SUB_NAME, ITEM_NAME, EXPEN_SEL, EXPEN_SEL명
)

-- Step 3: 매출 비율로 판매관리비 배부
INSERT INTO DOI_SMCE_COST
SELECT
    a.YYYYMM,
    'ACTUAL' AS sel_code,
    a.SITE,
    '양산' AS 구분,
    b.model,
    a.EXPEN_SEL명, a.SUB_NAME, a.ITEM_NAME, a.EXPEN_SEL,
    a.TOT_ACCT,      -- 판관비 총액
    b.SALE_AMT,      -- 모델별 매출
    b.TOT_AMT,       -- 전체 매출
    a.TOT_SMCE,      -- 판관비 합계
    b.DIST_RATE,     -- 배부비율
    ROUND((a.tot_acct * b.dist_rate), 0) AS DIST_AMT  -- 💰 배부금액
FROM sale_cntr a
LEFT JOIN sale_rate b ON (1=1)  -- Cross Join
ORDER BY b.model;
```

## 4. 수식 및 검증 로직

### 4.1 수량 수식 (DOI_STOCK)

```sql
-- ✅ 기본 수식
EOH = BOH + INPUT - OUT

-- 검증 쿼리
SELECT 
    YYYYMM, SITE, MODEL, STOCK,
    BOH, INPUT, OUT, EOH,
    (BOH + INPUT - OUT) AS CALC_EOH,
    EOH - (BOH + INPUT - OUT) AS DIFF
FROM DOI_STOCK
WHERE ABS(EOH - (BOH + INPUT - OUT)) > 0.01;
```

### 4.2 금액 수식 (DOI_STOCK_COST) - 총평균법

```sql
-- 1. 총 투입금액
총투입금액 = BOH_AMT + IN_AMT

-- 2. 총 수량
총수량 = BOH + INPUT

-- 3. 평균단가 (총평균법 - Moving Average)
평균단가 = 총투입금액 / 총수량
        = (BOH_AMT + IN_AMT) / (BOH + INPUT)

-- 4. EOH 금액
EOH_AMT = 평균단가 × EOH
        = ((BOH_AMT + IN_AMT) / (BOH + INPUT)) × EOH
        = (BOH_AMT + IN_AMT) / (EOH + OUT) × EOH
        
-- 단, OUT = 0인 경우:
EOH_AMT = BOH_AMT + IN_AMT  (전량 재고)

-- 5. OUT 금액 (차감법)
OUT_AMT = BOH_AMT + IN_AMT - EOH_AMT

-- ✅ 검증 수식
OUT_AMT = BOH_AMT + IN_AMT - EOH_AMT
```

**검증 쿼리:**

```sql
SELECT 
    YYYYMM, SITE, MODEL, 구분,
    
    -- 수량
    BOH, INPUT, OUT, EOH,
    
    -- 금액
    BOH_AMT, IN_AMT, EOH_AMT, OUT_AMT,
    
    -- 계산값
    (BOH_AMT + IN_AMT - EOH_AMT) AS CALC_OUT_AMT,
    
    -- 차이
    OUT_AMT - (BOH_AMT + IN_AMT - EOH_AMT) AS DIFF
FROM DOI_STOCK_COST
WHERE ABS(OUT_AMT - (BOH_AMT + IN_AMT - EOH_AMT)) > 0.01;
```

## 8. 주요 알고리즘

### 8.1 총평균법 (Moving Average) 원가계산

```
원리:
- 기초재고와 당월입고를 합산하여 평균단가 산출
- 출고 시 이 평균단가를 적용

계산 순서:
1. 기초재고 + 당월입고 = 총투입
   - 수량: BOH + INPUT
   - 금액: BOH_AMT + IN_AMT

2. 평균단가 산출
   평균단가 = (BOH_AMT + IN_AMT) / (BOH + INPUT)

3. 기말재고금액 계산
   EOH_AMT = 평균단가 × EOH
           = ((BOH_AMT + IN_AMT) / (BOH + INPUT)) × EOH

4. 출고금액 계산 (차감법)
   OUT_AMT = BOH_AMT + IN_AMT - EOH_AMT

검증:
   BOH_AMT + IN_AMT = EOH_AMT + OUT_AMT
```

### 8.2 단가 매칭 우선순위

```
Priority 1: 모델 + 구분(양산/개발) 완전 매칭
- DOI_STOCK.model = DOI_FAB_COST.model
- AND DOI_STOCK.구분 = DOI_FAB_COST.구분
- 조건: 'model_1'

Priority 2: 모델만 매칭 (구분 무시)
- DOI_STOCK.model = DOI_FAB_COST.model
- Priority 1에서 매칭 안된 경우만
- 조건: 'model_2'

Priority 3: inch 평균단가 사용 (모델 미매칭 시)
- DOI_STOCK.inch = DOI_FAB_COST.inch (그룹핑)
- inch별 평균단가 적용
- 조건: 'inch'
```

### 8.3 판매관리비 배부

```
배부 기준: 매출액 비율

계산:
1. 모델별 매출 비율 산출
   dist_rate = 모델별 매출 / 전체 매출

2. 판관비 항목별 배부
   DIST_AMT = 판관비 항목 금액 × dist_rate

특징:
- Cross Join으로 모든 모델에 배부
- 구분은 '양산'만 대상
```

IF @수량수식오류 = 0 AND @금액수식오류 = 0 AND @생산재고불일치 = 0
    PRINT '✅ 모든 검증 통과!';
ELSE
    PRINT '⚠️ 오류가 발견되었습니다. 상세 확인이 필요합니다.';

### 10.2 3단계 프로세스

```
Phase 1: 제품 BOH 금액 집계
├─ 입력: DOI_STOCK (수량), DOI_FAB_COST (단가, expen_sel)
├─ 로직: BOH 금액 = BOH 수량 × out_단가 (원가항목별)
├─ 출력: DOI_STOCK_BOH (expen_sel, expen_sel명 포함)
├─ 단가 매칭: model+구분 → model → inch 평균
└─ ⭐ 원가항목별 세분화: 재료비, 가공비 등 구분 집계

Phase 2: 제품 수불 금액 집계
├─ 입력: DOI_STOCK_BOH, DOI_FAB_COST.[out], DOI_STOCK
├─ 로직: 총평균법 (Moving Average)
│   - 평균단가 = (BOH_AMT + IN_AMT) / (BOH + INPUT)
│   - EOH_AMT = 평균단가 × EOH
│   - OUT_AMT = BOH_AMT + IN_AMT - EOH_AMT
├─ 출력: DOI_STOCK_COST
└─ 연결: DOI_FAB_COST.[out] = DOI_STOCK_COST.IN_AMT

Phase 3: 판매관리비 배부
├─ 입력: DOI_SALE_RESC (매출), DOI_ACCT_EXPEN (판관비)
├─ 로직: 매출 비율 배부
│   - dist_rate = 모델별 매출 / 전체 매출
│   - DIST_AMT = 판관비 × dist_rate
└─ 출력: DOI_SMCE_COST
```

### 10.3 주요 특징

1. **총평균법 원가계산**
   - 기초재고와 당월입고를 합산하여 평균단가 산출
   - 출고 시 평균단가 적용
   - 선입선출법(FIFO)이 아닌 총평균법 사용

2. **생산-재고 연결**
   - 가공비 배부의 OUT = 재고 수불의 IN
   - 데이터 정합성 핵심 검증 포인트

3. **단가 매칭 우선순위**
   - Priority 1: 모델 + 구분 완전 매칭
   - Priority 2: 모델만 매칭
   - Priority 3: inch 평균단가

4. **⭐ 원가항목별 세분화 (v1.1 추가)**
   - expen_sel 필드를 통한 원가요소별 집계
   - 재료비, 가공비, 노무비 등을 구분하여 BOH 금액 산출
   - 더 정교한 원가 분석 및 관리 가능
   - DOI_STOCK_BOH 테이블에 expen_sel, expen_sel명 컬럼 추가

5. **트랜잭션 관리**
   - BEGIN TRANSACTION ~ COMMIT/ROLLBACK
   - 에러 발생 시 자동 롤백
   - 상세 로그 메시지 제공

### 10.4 주의사항

⚠️ **실행 전 확인 사항:**
- C0003005 (가공비 배부) 선행 완료 필수

- DOI_FAB_COST 데이터 존재 확인
